1. What is Semantic HTML? 
Semantic HTML gives HTML meaning and basic description of the part so that the someone who is unfamiliar with code can look at it and be able to tell what it is. Ie: a <section> is a section of a website.

2. What is HTML used for? 
HTML is used for marking up a website to give the basic structure to a website. 

3. What is an attribute and where do we put it? 
An attribute provides additional information about an element. You put it after an element, such as <img <-'this is the element' src<-'this is the attribute.

4. What is the h1 tag used for? How many times should I use it on a page?
The h1 tag is the Heading 1 tag. It is the biggest of the heading tags, and denotes the most important info on the website. You should use h1 once on a page.

14. What is the alt attribute used for? 
The alt attribute gives text for an image when the image can't viewed. either by web problems or if someone is using a screen reader.

15. How do I reference an id?

example: <section id="example"> ----> #example{}

16. What is the difference between a section and a div
<section> defines a section of the document/website. It is semantic. <div>s aren't semantic, and define divisions in the content of the document or the webpage.

19. What is the difference between a class and an id? - Give me an example of when I might use each one
The difference between a class and id aside from being called differently class -> .class, id ->#"id" is in the specificity amount. id's are more specific than classes.
you would use a class when you want consistant styling of multiple elements. you would use an id when you have one specific item/element on the website that gets that style.
use class styling for things like sections, and ids for things like headers or main content.

22. What is a parent child selector? When would this be useful? 
a parent child selector is a selector that selects all elements of a direct child of the elment. This would be useful when styling multiple child elements because you would have to write it once and it would continue to apply to those children.

37. Walk me through Lambda's git flow. 
first you fork the repo, make the collaborator your TL, then clone the file and download it to your local computer, cd into the file, create a branch, work on the repo to get it done,  git add ., git commit the project with a message., then push the changes back up to github with git push -u origin "branch-name", and then make a pull request.
